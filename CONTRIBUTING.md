# Contributing Guidelines

Thank you for your interest in contributing to this project.
This repository is intended to support reproducible research and
open-source experimentation in phishing detection.

---

## How to Contribute

Contributions are welcome in the following areas:
- Feature engineering and validation
- Model benchmarking and evaluation
- Threat score calibration
- Tooling and deployment improvements
- Documentation and reproducibility enhancements

---

## Getting Started

1. Fork the repository
2. Create a new branch for your changes
3. Install dependencies using:
   ```bash
   pip install -r requirements.txt
4. Ensure notebooks and scripts run end-to-end
5. Submit a pull request with a clear description

##Coding & Research Standards

- Prefer clarity and interpretability over optimization
- Document assumptions and limitations explicitly
- Avoid introducing unnecessary dependencies
- Ensure results are reproducible

## Issues & Discussions
- Use GitHub Issues for bugs, enhancements, or research proposals
- Clearly scope issues and expected outcomes
- Discussions should focus on technical justification

## Code of Conduct 
Be respectful, constructive, and professional.
This project follows standard open-source collaboration norms.
